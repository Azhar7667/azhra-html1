الدرس الثاني: التعاون في Git (Collaboration)

الدرس الثاني: التعاون في Git

في هذا الدرس سنتعلم كيفية العمل بشكل تعاوني على المشاريع باستخدام Git وGitHub، وكيفية إدارة الفرق والمشاركة في تطوير البرمجيات.

خطوات العمل التعاوني

1
إنشاء مستودع على GitHub ومشاركته مع فريق العمل
2
نسخ المستودع إلى الجهاز المحلي باستخدام git clone
3
إنشاء فرع جديد لكل ميزة أو تصحيح يتم العمل عليه
4
رفع التغييرات وإنشاء طلبات السحب (Pull Requests)
5
مراجعة الكود ودمج التغييرات مع الفرع الرئيسي

أوامر Git للتعاون

  • git clone: نسخ مستودع من GitHub إلى الجهاز المحلي
  • git branch: إنشاء وعرض وإدارة الفروع
  • git checkout: الانتقال بين الفروع
  • git pull: جلب التحديثات من المستودع البعيد
  • git push: رفع التغييرات إلى المستودع البعيد
  • git merge: دمج الفروع مع بعضها
  • git fetch: جلب المعلومات من المستودع البعيد دون الدمج
  • إضافة مساهمين: إدارة صلاحيات الفريق في GitHub
